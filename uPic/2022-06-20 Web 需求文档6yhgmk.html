1 全局说明


1.1 文档内容


当产品文档与设计稿的细节有冲突时，以产品文档为准。

  • 例如产品文档中说明的提示文案和设计稿文案，以产品文档为准。
  • 如果文档中有说明「以设计稿」为准的，以设计稿为准。
  • 未做说明的，以产品文档为准。

1.2 登录与未登录


  • 未登录：游客模式，部分页面功能会有所限制。
  • 已登录未通过高级验证：部分页面功能会有所限制。
  • 已登录已通过高级验证：页面功能无限制。

1.3 多端在线


不允许两个 Web 同时在线，Web 唯一的判断标准：设备+浏览器；例如：

  • 在 Mac 的 Chrome 登录后，不允许在其他设备重复登录；如果重复登录了，则之前登录信息会失效，并恢复为未登录状态。
  • 在 Mac 的 Chrome 登录后，不允许在该 Mac 的其他浏览器重复登录；如果重复登录了，则之前登录信息会失效，并恢复为未登录状态。

允许一台手机和一个 Web 同时在线，例如：

  • iPhone + Web。
  • Android + Web。

不允许两台手机同时在线，例如：

  • iPhone + iPhone。
  • iPhone + Android。
  • Android + Android。

1.4 组件逻辑-日期选择器


参考 Element-Date Picker-选择一段时间-带快捷选项。

  • 定义：以日为单位，在一个选择器中便捷地选择一个时间范围。
  • 快捷选项：选择最近的一周；最近的一个月；最近的三个月。
  • 默认显示日期：无，即显示所有时间段的数据。
  • 显示清除按钮：清除选项后，恢复到默认选项。
  • 选择时间范围，即时刷新表格数据，即无需点击查询按钮。
  • 占位符：请选择-请选择。


1.5 组件逻辑-输入框


  • 类型：通过鼠标或键盘输入字符。
  • 占位文本：请输入。
  • 一键清空：使用 clearable 属性即可得到一个可一键清空的输入框。
  • 当输入时，自动补全：根据当前输入的内容获取对应的输入建议。、
  • 远程搜索：从服务端搜索数据。


1.6 组件逻辑-选择框


参考 Element Select 选择器-筛选选项组件。

  • 类型：Select 选择框。
  • 输入长度：不限制。
  • 占位文本：请选择。
  • 可清空单选：包含清空按钮，可将选择器清空为初始状态。
  • 可以利用筛选功能快速查找选项。
  • 多选。


1.7 组件逻辑-分页


参考 Element-Date Picker-选择一段时间-带快捷选项。

  • 显示当前页数。
  • 只有一页时是否隐藏分页器。
  • 每页条数默认 20 条；可更改。
  • 支持快速跳转至某页。
  • 显示最大页码按钮数。


1.8 组件逻辑-表单


  • 定义：允许用户在文本域、下拉列表、单选框、复选框等等）输入信息的元素，例如「注册登录」「币币交易」「弹窗」等页面中都有表单。
  • 表单在必填写必选项，未填写未选择时，提交按钮置灰，不可点击。
  • 只有当必填写必选项，全部填写完成时，提交按钮才可点击。
  • 点击后的逻辑各不相同，见具体说明。

1.9 组件逻辑-表格


参考「Element-Table-带斑马条纹表格」。

  • 排序：默认按照时间倒序排序。
  • 显示数量：默认一页 20 行数据；超过显示分页器。
  • 带斑马纹表格：使用带斑马纹的表格，可以更容易区分出不同行的数据。
  • 单元格长度根据字段值长度自适应，超过固定宽度后，在底部显示横向浏览指示器。


1.10 组件逻辑-Popconfirm


  • 定义：点击某个元素弹出一个简单的气泡确认框，用于关键操作的二次确认。
  • 点击确认按钮时，确认操作。
  • 点击取消按钮，否定操作，并关闭。


1.11 组件逻辑-提示


参考「Element-Message-基础用法」。

  • 定义：常用于主动操作后的反馈提示。 与 Notification 的区别是后者更多用于系统级通知的被动提醒。
  • 显示从顶部出现，3 秒后自动消失。
  • 不同状态：用来显示「成功、警告、消息、错误」类的操作反馈。


1.12 数据格式


  • 数值：默认使用千分位帮助用户阅读；例如 123,220。
  • 日期：YYYY-MM-DD。
  • 时间：HH:MM。
  • 日期和时间：YYYY-MM-DD HH:MM。
  • 百分比：比例问题时使用百分比；精确到小数点后两位。
  • 银行卡号码：保留前四位与后六位，其余 * 表示。仅能识别发卡行与大部份个人账号标识。

位置排列

  • 在表格中，诸如金额、数量等数值分布排列时，通常采用「右对齐」方式，既方便用户快捷读取数据，还可以使用户进行纵向数据对比。


数字脱敏

  • 数据脱敏是指对某些敏感信息通过脱敏规则进行数据变形，实现敏感隐私数据的可靠保护。此处给出的脱敏规则为通用产品规范，遇到数据安全性较强的业务场景，可根据业务场景自行调整。


无数据时用 - - - 表示。



数据加载

  • 数据加载用「骨架屏」表示。

2.7 邀请码


邀请码可在 MTP 后台配置：

  • 对有所人生效：在「注册登录」页面是否显示邀请码字段。默认不显示。
  • 对有所人生效：在「顶部导航栏」是否显示邀请码入口。默认不显示。
  • 对某些人生效：谁有邀请码。
  • 邀请码所有人都可以填。

邀请码逻辑：

  • 邀请码仅可在首次注册账户时使用。您或受邀人必须输入正确的推荐码。如果受邀人在注册时忘记使用或输入了错误的推荐码，则您将无法赚取该受邀人产生的佣金。
  • 您以在受邀用户的币币交易、差价合约、期权进行交易时赚取佣金。
  • 推荐人只有先自己有对应的账户，即币币交易账户、差价合约账户、期权账户，才能从好友交易中赚钱。
  • 推荐佣金将实时计算，每间隔一小时转账至相应的币币帐户。只有在受邀人完成交易后，您才能获得推荐佣金。

3.2 手机注册


手机号码：

  • 必填。
  • 输入框分为两部分：第一部分是「国际电话区号 」，第二部分是「手机号码」。
  • 国际电话区号：根据网站的语言设置决定：
    • 简体中文：默认填写 +86。
    • 繁体中文：默认填写 +852。
    • 英文：默认填写 +1。
    • 可删除默认填写的区号，并自行填写；只能删除数字，+ 号无法删除。
    • 只能输入数字，最多三位数。
  • 手机号码：只能输入数字；最少输入位数不限制；最多 11 位数。

验证码按钮：

  • 必填。
  • 有「获取验证码」和「N 秒后重发」两种状态。
  • 获取验证码：可点击，点击后将验证码发送至已填写的手机号，同时状态改为「N 秒后重发」。
  • N 秒后重发：按钮置灰，不可点击。N 为 60 秒倒计时，倒计时结束后，状态改为「获取验证码」

机构代码：

  • 必填。
  • 机构代码由 MTP 后台设置， 默认为设置的默认值。
  • 默认值可更改。
  • 字符输入限制：字母（大小写不限）、数字和符号（仅限 ASCII 标准字符）。

设置密码：

  • 必填。
  • 字符长度：8-16 位。
  • 字符输入限制：字母（大小写不限）、数字和符号（仅限 ASCII 标准字符）。
  • 提示文案：请使用 8-16 个字母、数字和符号的组合。
  • 密码小于 8 位时，提示：密码小于 8 位，请重新输入。
  • 密码大于 16 位时，提示：密码大于 16 位，请重新输入。
  • 密码只有数字时，提示：密码中未包含字母和符号，请重新输入。
  • 密码只有字母时，提示：密码中未包含数字和符号，请重新输入。
  • 密码只有符号时，提示：密码中未包含数字和字母，请重新输入。
  • 密码只有数字和字母时，提示：密码中未包含符号，请重新输入。
  • 密码只有数字和符号时，提示：密码中未包含字母，请重新输入。
  • 密码只有字母和符号时，提示：密码中未包含数字，请重新输入。

4.2 流程概述


卖家发布「我想卖广告」，买家买：

  • 卖家先发布卖币的广告，此时平台在「法币账户」冻结对应的币种数量；若无币种数量，则无法发布广告。
  • 卖币广告显示在「买币-购买」列表。
  • 买家看到，点击「购买」并用法币付款。
  • 卖家收到法币后，确认放行。
  • 买家「法币账户」增加对应的余额，卖家「法币账户」扣除对应余额。

买家发布「我想买广告」，卖家卖：

  • 买家先发布买币的广告。
  • 买币广告显示在「买币-出售」列表。
  • 卖家看到，点击「出售」并在「法币账户」冻结对应的币种数量；若无币种数量，则无法出售。
  • 买家看到有人卖后，点击「购买」并用法币付款。
  • 卖家收到法币后，确认放行。
  • 买家「法币账户」增加对应的余额，卖家「法币账户」扣除对应余额。

交易流程及场景：

  • 买家在规定时间内付款，卖家在规定时间内收款并放行，交易完成。
  • 买家在规定时间内付款，卖家未在规定时间内收款并放行，交易超时，自动完成。
  • 买家在规定时间内付款，但实际上没付款，卖家没有收到，卖家申诉，人工介入。判定买方违约，取消卖家冻结占用资金；判定卖方违约，自动放行。
  • 买家未在规定时间内付款，交易超时，订单关闭。
